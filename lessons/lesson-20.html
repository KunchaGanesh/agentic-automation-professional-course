Data-Driven Routing & Workflow Decisions

Driving Workflow Intelligence: How Data

Shapes Automated Decisions in Maestro

Gateways and Conditional Logic in Maestro

Gateways in Maestro act as decision points that determine how a workflow proceeds based on data. The two primary types—Exclusive and Parallel Gateways—enable you to build flexible, responsive processes. Exclusive Gateways evaluate conditions and route the process down a single path, while Parallel Gateways allow multiple paths to execute simultaneously, synchronizing them at a later point. Both types rely on process variables to make informed, automated decisions that drive workflow efficiency.

Types of Gateways and Their Functions

Expand each section to learn about the main gateway types and how they shape workflow routing in Maestro.

Exclusive Gateway

An Exclusive Gateway evaluates a set of conditions and routes the process down exactly one path—the first condition that evaluates to true. Each outgoing path from a gateway has an associated condition expression. Conditions use standard comparison and logical operators: ==, !=, >, <, >=, <=, &&, ||. A default path can be configured as a fallback if no conditions match.

This type of gateway is essential for making clear, rule-based decisions in your workflow, ensuring only the correct branch is followed based on the data provided.

Parallel Gateway

Parallel gateways allow multiple paths to execute simultaneously and synchronize at a joining gateway. When a parallel gateway is reached, all outgoing paths are activated at the same time, enabling concurrent processing of tasks or activities. Parallel gateways are useful when different parts of a process can occur independently but must eventually come together before the workflow continues.

Writing Condition Expressions: Data-Driven Routing

Gateways use the Expression Editor to evaluate process variables and determine which path a workflow should take. By writing clear condition expressions, you can

automate complex routing logic based on real-time data. It’s important to use the correct operators and ensure variables are in the appropriate scope—typically global —so gateways can access them. Well-crafted conditions are the key to creating workflows that branch intelligently and adapt to changing business requirements.

Explore Common Condition Expressions

Expand each tab to see how different operators and configuration options shape your workflow’s decision logic.

Comparison operators are used to evaluate the relationship between process variables and specific values. Common operators include == (equals), != (not equals), > (greater than), < (less than), >= (greater than or equal to), and <= (less than or equal to). For example, a condition like PO_Amount <= 1000 checks if the purchase order amount is less than or equal to 1000. These operators help define precise routing rules for each gateway path.

Logical operators allow you to combine multiple conditions within a single expression. The && operator means "and," requiring all conditions to be true, while the || operator means "or," requiring at least one condition to be true. For instance, PO_Amount > 1000 && PO_Amount <= 5000 routes only when both conditions are met. Logical operators make it possible to build more nuanced and flexible routing logic.

Configuring a default path in a gateway ensures that the workflow has a safe fallback if none of the specified conditions are met. This prevents the process from stalling due to unexpected data values. Always set a default path as a safety net, especially in complex workflows where not all possible data scenarios can be anticipated. This is a critical best practice for robust automation design.

Best Practices for Data-Driven Routing

Explore these essential steps to ensure your workflow routing is robust, maintainable, and fully automated.

Variable Preparation

Step 1: Collect data via an App form or agent and store in global variables. This ensures all routing logic has access to the necessary information throughout the process. Proper variable preparation is the foundation of reliable workflow decisions.

Gateway Evaluation

Step 2: Exclusive Gateway reads variable values and routes accordingly. Gateways should always reference up-to-date global variables for accurate decision-making. Careful configuration of gateway conditions ensures the workflow follows the correct path every time.

Task Output Updates

Step 3: Task outputs update global variables for subsequent routing. After each task, make sure any new data or decisions are reflected in the process variables. This keeps the workflow state current and ready for the next decision point.

Notification Personalization

Step 5: Notification tasks use variables to personalize messages. By referencing process variables, you can tailor communications to each requester or stakeholder. Personalized notifications improve user experience and process transparency.
