Deploying and Integrating Coded Agents with UiPath Orchestrator

Bridging Code and Enterprise: Deploying Your Coded Agents

Seamless deployment and integration are the linchpins of agentic automation in the enterprise. Moving from code to production is more than a technical step—it’s the transformation of your coded agents into auditable, scalable, and governed solutions that can be trusted in mission-critical workflows. UiPath Orchestrator serves as the bridge, enabling packaged agents to operate securely, be monitored in real time, and integrate with the broader automation ecosystem. Mastering this process ensures your automation is not only powerful, but also compliant and enterprise-ready.

By completing this lesson, you will gain the skills needed to effectively deploy and manage coded agents within UiPath Orchestrator, as well as integrate them with various Orchestrator features and apply best practices for automation governance and scheduling.

What you’ll learn

  1. Package coded agents for deployment to UiPath Orchestrator.
  2. Publish and manage .nupkg packages within Orchestrator’s package feeds.
  3. Create and configure agent processes in Orchestrator folders.
  4. Integrate agents with UiPath assets, storage buckets, and other processes.
  5. Apply governance and scheduling best practices for reliable automation.

Deployment is where code meets compliance—transforming your logic into enterprise-grade automation that’s secure, traceable, and ready for scale.

Packaging Coded Agents for Orchestrator

The uipath pack command takes your code and packages it into the standard .nupkg format that UiPath uses for automation packages. Using a UiPath SDK, you can package this code and deploy it to Orchestrator as a standard .nupkg package in a package feed of your choice.

You can assign an explicit version using the --version flag, or use --auto-version for automatic versioning. You can also embed VCS metadata for traceability using the -repository-url flag. The package key format in Orchestrator is PackageId:Version, which helps track and manage versions across feeds.

Publishing .nupkg Packages

Upload your .nupkg package to Orchestrator using the web interface or CLI. Select the appropriate package feed to control visibility and access. Ensure the package is correctly named and versioned for easy identification. After publishing, verify that the package appears in the feed and is available for process creation.

Version Control and Feeds

Maintain multiple versions of your agent packages to support staged releases, rollbacks, and testing. Use feeds to separate development, staging, and production environments, ensuring only validated packages reach critical workflows. Careful version management prevents accidental overwrites and supports compliance requirements.

Validation and Troubleshooting

  • Always validate your package in a test environment before deploying to production. Use
  • Orchestrator logs and error messages to troubleshoot issues quickly.
  • Have a rollback plan in place, and document known issues and resolutions for future reference.

Managing Agent Processes within

Orchestrator Folders

Once published, your coded agent appears in Orchestrator as a regular package. You can create a process from it by selecting the package and assigning it to a specific folder. Folders have their own Orchestrator resources, and only those available in the active folder are visible in Orchestrator.

To manage access within a folder, you assign folder roles that include the necessary permissions. A folder hierarchy can be established with up to 7 levels, and user access is inherited from parent folders down to all subfolders. For agent monitoring, navigate to Monitoring > Agents to view live instances, completed instances, errors, and agent unit consumption — filtered by interval, agent, or folder.
