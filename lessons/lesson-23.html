Configuring & Managing Autopilot for Business Users

Empowering Business Users with AI: The

Role of Autopilot in UiPath

Autopilot for Everyone is transforming how business users interact with automation and organizational knowledge. By providing AI-powered assistance directly within the UiPath Assistant, Autopilot eliminates technical barriers and empowers employees to access automations, insights, and company-specific information through natural language. Configuring and managing Autopilot effectively is essential for both administrators and end users, ensuring that every employee can leverage automation and AI to drive productivity and informed decision-making.

In this lesson, you will gain a comprehensive understanding of how to effectively use and manage Autopilot for Everyone. The following objectives outline the key skills and knowledge areas you will develop:

What you’ll learn

  1. Describe the four core capabilities of Autopilot for Everyone.
  2. Configure the base (general-purpose) Autopilot experience.
  3. Create and manage specialized Autopilots for focused business functions.
  4. Select and configure tools (automations) and indexes for each Autopilot.
  5. Understand and manage context reversion and session behavior.
  6. Apply best practices for deployment and ongoing management.
  7. Test your knowledge with practical scenario-based checks.

Autopilot for Everyone: Core Capabilities Explained

Autopilot for Everyone is UiPath’s AI assistant designed specifically for business users, not developers. It enables employees to interact with automation, organizational knowledge, and generative AI using natural language—no technical expertise required.

The four core capabilities are:

  • Running automations by describing tasks in plain English
  • Asking questions and generating AI-powered insights
  • Querying company data through Context Grounding
  • Using Clipboard AI for intelligent copy-and-paste between applications

Autopilot is not intended for building new automations; its focus is on running and interacting with existing ones to streamline daily business tasks.

Configuring the Base Autopilot Experience

The general Autopilot for Everyone is configured at the tenant level in Automation Cloud. Administrators determine which automations and knowledge indexes are available by selecting the appropriate Orchestrator folder. This base configuration provides broad access to all tools and indexes the user has permission for, without restricting the scope to a specific business function. Specialization, on the other hand, allows for focused experiences by limiting access to only the most relevant automations and knowledge for a particular use case.

Creating and Managing Specialized Autopilots

Specialized Autopilots are customized AI assistants designed for specific business functions such as HR, Finance, or IT Support. They provide focused, domain-specific guidance using custom system instructions. Administrators can configure relevant Context Grounding indexes for accurate responses. Only the required tools and automations are available to users. Suggested starting prompts help users quickly begin common tasks. This creates a streamlined, secure, and business-focused AI experience.

Introduction: Setting Up a Specialized Autopilot

Follow these steps to set up a focused Autopilot for a specific business function.

  1. 1
    Access Autopilot Configuration
    Navigate to Automation Cloud → Admin → AI Trust Layer → Autopilot for Everyone.
  2. 2
    Select Tenant
    Choose the tenant where Autopilot for Everyone is installed.
  3. 3
    Create Specialized Autopilot
    Click Create a specialized Autopilot and enter the display name, description, Orchestrator folder, and custom system prompt.
  4. 4
    Save General Configuration
    Save the initial configuration to create the specialized Autopilot.
  5. 5
    Configure Starting Prompts
    Add suggested prompts to help users quickly start common tasks.
  6. 6
    Enable Context Grounding Indexes
    Select the appropriate knowledge indexes to provide accurate, domain-specific responses.
  7. 7
    Configure Tools and Automations
    Choose the tools and automations that the specialized Autopilot can access.

Review and Finalize

Double-check all settings to ensure the specialized Autopilot is focused, helpful, and ready for deployment.

Integrating Tools and Automations with Autopilot

Integrating Tools and Automations with Autopilot

Autopilot allows users to trigger automations using natural language. Administrators configure the tools and automations available to each Autopilot. Supported tools include UiPath processes, API workflows, and platform capabilities. Specialized Autopilots expose only the tools relevant to a specific business function. Clear display names and descriptions help Autopilot identify the correct automation. This improves accuracy, reduces confusion, and delivers a better user experience.

Using, Switching, and Deploying Specialized Autopilots

Users can switch between the general Autopilot and specialized Autopilots based on their needs. Selecting a specialized Autopilot loads its configured tools and knowledge indexes. Users can switch between Autopilots during a conversation without interruption. Starting a new chat resets the session to the general-purpose Autopilot. Administrators should monitor usage and gather feedback after deployment. Regular updates and refinements help improve accuracy and user experience.

Introduction: Interacting with Multiple Autopilots

See how users interact with multiple Autopilots in a typical session.

  2. 2

    Choose a specialized Autopilot (such as HR or IT Support) from the dropdown.
  3. 3
    Switch Mid-Conversation
    Switch to another specialized Autopilot as your requirements change.
  4. 4
    Context Reversion
    Begin a new chat to return to the general-purpose Autopilot with a fresh session.

Flexible, User-Centric Experience

Users can easily switch between the general and specialized Autopilots based on their current task. Each specialized Autopilot automatically loads the appropriate tools and knowledge indexes.

This ensures users always have access to the most relevant capabilities. The experience remains seamless as users move between different business functions. New chat sessions start with the general-purpose Autopilot by default. This approach provides a flexible, efficient, and user-focused experience.
