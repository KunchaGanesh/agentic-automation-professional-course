Developing Coded Agents with Python and UiPath SDKs

From Concept to Code: Building EnterpriseReady Coded Agents

Now that you understand the foundations of agentic automation, it’s time to get hands-on. In this lesson, you’ll learn how to move from high-level concepts to practical development—building coded agents using Python and UiPath SDKs. Mastering these tools is essential for creating robust, flexible, and maintainable agents that drive enterprise automation at scale. Whether you’re new to Python automation or looking to deepen your expertise, this lesson will equip you with the skills to turn automation ideas into production-ready solutions.

In this lesson, you will gain the foundational skills needed to create, manage, and deploy agents using Python and UiPath SDKs. The following objectives outline what you will accomplish by the end of this lesson:

What you’ll learn

  1. Set up a Python development environment for agent creation.
  2. Define agent logic and orchestration flows using UiPath SDKs.
  3. Utilize orchestration primitives and integrate UiPath resources.
  4. Package and version agents for deployment.
  5. Apply best practices for modular and maintainable code.

Getting Started with Python and UiPath SDKs for Coded Agents This guide will walk you through the essential steps to start developing coded agents using Python and UiPath SDKs. By following these steps, you'll be able to efficiently create, test, and prepare agents for enterprise deployment.

  1. 1
    Prepare Your Development Environment
    Set up Python 3.11 or higher with either pip or uv installed, and install the UiPath CLI globally via npm (npm install -g @uipath/cli). Choose an IDE like VS Code or JetBrains that supports Python development with debugging and code completion. Ensure you have authenticated with UiPath Automation Cloud by running uipath auth in your project directory.
  2. 2
    Initialize a New Agent Project
    Use uip agent init to scaffold a new agent project, which creates the structure and configuration files needed for your agent. The Python SDK enables creation, packaging, and deployment of coded agents
  3. 3
    Agent Logic and Behavior
    Write the core Python code for your agent using frameworks like LangGraph or LlamaIndex. Define the agent's reasoning patterns, decision-making logic, and tool usage. Include interrupt points within your agent to ask for human input when needed, allowing for human-in-the-loop scenarios where the agent execution pauses and a task is created in Action Center.
  4. 4
    Integrate UiPath Platform Resources
    Use UiPath SDKs to invoke and interact with other UiPath resources directly within your code—including assets for secure credential storage, storage buckets for data access and manipulation, and connections to various systems and services. This enables seamless enterprise integration
  5. 5
    Test and Debug Locally
    Run debug tests and step-through debugging from Studio Web, or execute evaluations to validate agent behavior. Optionally, configure LangSmith environment variables (LANGCHAIN_TRACING_V2, LANGCHAIN_API_KEY, LANGCHAIN_PROJECT) to enable detailed tracing of your agent's execution for comprehensive debuggin
  6. 6
    Package and Version Your Agent
    Use the uipath pack command to package your code into the standard .nupkg format that UiPath uses for automation packages. Then use the uipath publish command to upload your packaged .nupkg file to Orchestrator's package feed. Assign semantic versions and document changes for production tracking

Summary: Ready for Enterprise Deployment

By following these steps, you’ll be able to create, test, and prepare coded agents that are ready for enterprise deployment and integration.

Defining and Structuring Agent Logic for Scalable Automation

Learn how to design agent workflows for maintainability and scalability by leveraging code-based logic, orchestration frameworks, and modular code organization.

Agent Logic Fundamentals

Coded agents allow you to define agent behavior directly in Python code, giving you complete control over reasoning patterns, tool usage, and decision-making logic. Using frameworks like LangGraph or LlamaIndex, you write code that outlines the exact steps and decisions your agent takes - much like designing a sequence of activities in UiPath Studio. Unlike traditional workflows that follow fixed paths, agents dynamically plan actions based on intermediate results, selecting tools, interpreting results, and adjusting behavior based on current context and memory. You can define interrupt points within your agent to ask for human input when needed, allowing the agent to pause execution and create tasks in Action Center for human review or decision-making.

By structuring your code and logic clearly- with well-defined prompts, tool descriptions, and execution patterns- you ensure your agent is both powerful and easy to maintain as requirements evolve.

Orchestration Flows with LangChain & LlamaIndex

LangGraph (part of the LangChain ecosystem) enables graph-based orchestration, where agent behavior is modeled as interconnected nodes and edges. Nodes represent computational steps (tool calls, LLM invocations, data processing), while edges define transitions and conditional routing logic based on the current state. This approach allows for advanced flows such as memory checkpoints that persist agent state across interactions, conditional branching that dynamically routes execution based on runtime conditions, and human-in-the-loop interruptions that pause execution and create tasks in Action Center for human review or decision-making. LlamaIndex complements this by providing specialized capabilities for retrieval-augmented generation, allowing your agents to access and reason over large document corpora with high precision. By combining these frameworks - LangGraph for orchestration and state management, and LlamaIndex for intelligent data retrieval—you can build agents that handle sophisticated workflows and adapt dynamically to enterprise needs.

Modular Code Structure

Keep each agent focused on a single responsibility - avoid overloading agents with multiple roles. Use environment variables to separate configuration from code, making updates easier across environments. Apply inheritance and modular patterns to avoid duplicating common code. Implement comprehensive testing using structured evaluation sets to test edge cases. Use the UiPath CLI to run validation, unit tests, and workflow analyzer checks during development. Sample projects in the UiPath GitHub repository demonstrate professional code organization patterns.

Integrating UiPath Platform Resources in Your Agent

UiPath SDKs provide developers with the necessary logic to invoke and interact with other UiPath resources directly within their code. The Python SDK provides programmatic access to UiPath Platform resources such as processes, assets, buckets, and jobs.

Developers can use UiPath assets for secure storage and management of sensitive data, access and manipulate data stored in UiPath storage buckets, and establish and manage connections to various systems and services. Additionally, developers can trigger and interact with other UiPath processes from within the coded agent.

Secret-type assets store secrets such as API keys. You can use secret-type assets only in conjunction with coded agents. All asset types are encrypted in the Orchestrator database by default.

You can configure your project with environment variables using secret assets, eliminating the need for direct code modifications. UiPath

Storage buckets provide a per-folder storage solution. Access to storage buckets and their contents is controlled using the fine-grained permissions and role assignment models of the Folders feature.

Introduction to Packaging and Versioning Your Agent

  1. 1
    Package Using the CLI
    The uipath pack command takes your code and packages it into the standard .nupkg format that UiPath uses for automation packages. The uipath publish command then uploads your packaged .nupkg file to Orchestrator's package feed.
  2. 2
    Version the Package
    You can assign an explicit version using the --version flag when running the pack command (for example: --version 1.4.0). The --auto-version option is also available if you prefer automatic versioning.
  3. 3
    Publish and Deploy
    Once published, your coded agent appears in Orchestrator as a regular package. You can then create a process from it within a specific folder. The agent process is governed by the same principles as a standard process — ready to be scheduled, triggered, executed, and integrated into broader automation workflows.

Summary: Ensuring Smooth Deployment

By following these steps, your agent will be properly packaged, versioned, and documented, supporting smooth deployment and effective enterprise governance.
